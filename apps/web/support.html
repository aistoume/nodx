nodx Lens · Chrome extension

How do I use it?

After installing, open Options (right-click the nodx Lens icon → Options) and paste your API key from one of: Anthropic, OpenAI, or Google. Then on any webpage, highlight some text — a small 🔍 button appears. Click it for an inline explanation.

The 🔍 button doesn't show up

Check three things, in order:

  • Is the API key saved in Options? Some pages (like new tab, Chrome Web Store, chrome:// pages) block extensions — try on a regular webpage.
  • Is your selection between 2 and 500 characters? Very short or very long selections are ignored to avoid noise.
  • In Options → Trigger, is "Show the explain button" turned on?
I get "Missing API key" or authentication errors

Open Options and paste your key again. Then reload the webpage — content scripts hold the old settings until reload. If your key was revoked or hit a rate limit, that message comes straight from the AI provider.

Do you see the text I highlight?

No. The text goes directly from your browser to the AI provider you configured. nodx Lens has no server. See the Privacy Policy for details.

Does it work in Chinese / Japanese / other languages?

Yes — the AI provider handles the language. In Options you can pick "Follow browser" or force Chinese / English for the UI itself.

nodx desktop · macOS app

How do I install and set up?

Download the .dmg from aicon.solutions/nodx, drag nodx.app to /Applications, right-click → Open (the app is signed but not Apple-notarized yet, so Gatekeeper prompts once). Then open ⚙ Settings and either:

  • Paste your Anthropic API key (sk-ant-…), or
  • Switch to Claude Code subscription mode if you already have claude CLI installed and logged in.
⌥+E doesn't do anything in other apps

Almost always a missing Accessibility permission. Open ⚙ Settings → "⌥+E 全局划词解释" → click "Open System Settings · Accessibility", then check the box next to nodx. Come back to nodx, click "Re-check" in Settings, and try again.

Claude Code mode says "claude not found"

macOS apps launched from Finder don't see your shell's PATH, so claude installed via nvm / bun / homebrew may be invisible to nodx. Either:

  • Run which claude in your terminal, symlink it: sudo ln -s $(which claude) /usr/local/bin/claude
  • Or start nodx from a terminal with CLAUDE_BIN="$(which claude)" open /Applications/nodx.app
How do I export or move my data?

All data is in a local SQLite database at ~/Library/Application Support/nodx.desktop/. From within nodx you can also export any topic subtree as a .nodx data pack (Document page header → "📦 Export") and re-import it on another machine.

General

How do I report a bug?

Email contact@aicon.solutions with:

  • The product and version (visible in Options / Settings)
  • What you did and what happened
  • What you expected to happen
  • A screenshot if it helps

For nodx desktop, if it's a crash, macOS keeps a crash report at ~/Library/Logs/DiagnosticReports/ — attaching that speeds up diagnosis a lot.

How do I request a feature?

Same email. Extra points if you explain the problem you're trying to solve — sometimes there's already a way, or a simpler approach.

Are Aicon products open source?

The website source (this page included) is at github.com/aistoume/aicon-web and is public. The product source repositories are currently private, but we're planning to gradually open-source portions as the products stabilise.

Do you offer refunds?

Our products are currently free (bring-your-own API key). When paid tiers ship (nodx Team / Enterprise), refund terms will be published here.
